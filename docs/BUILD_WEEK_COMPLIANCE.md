## Accuracy requirements

- Describe only implemented or demonstrated features.
- Keep untested multiuser, provider and live-media functions labeled as roadmap.
- Do not claim tests passed unless a successful run is available.
- Do not claim the APK is ready until the release file downloads correctly.
- Use the project name **Exovia NeuroCanvas** consistently.
- Credit third-party tools and dependencies honestly.
- Use only assets and content the team is permitted to publish.
- Keep private user and customer information out of the repository and submission.

## Judging alignment

The official OpenAI page lists these judging areas:

1. technical implementation;
2. design and user experience;
3. potential impact;
4. quality of the idea;
5. thoughtful use of GPT-5.6 and Codex;
6. clarity of the problem, solution and approach.

The project description and demo should show clear evidence for each area.
