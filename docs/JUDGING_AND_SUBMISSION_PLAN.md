# Judging and Submission Plan

## Official target

OpenAI Build Week evaluates projects on:

1. Technological implementation
2. Design and user experience
3. Potential impact
4. Quality of the idea

Strong submissions must also demonstrate thoughtful use of GPT-5.6 and Codex and clearly communicate the problem, solution and approach.

## Strategic positioning

Exovia NeuroCanvas should be entered in **Developer Tools**.

Core positioning:

> Exovia NeuroCanvas is a visual, auditable operating layer for knowledge and agent activity. It combines infinite semantic navigation, exact source evidence, compact ExiaL pulses, EXIR-style canonical events, safe Exil intent previews and FAPI capability maps.

This is stronger than presenting the project as a mind map, document viewer or agent dashboard.

## What judges should understand in the first 20 seconds

- The interface is working.
- The map is generated from real source content.
- Every visual node remains traceable to evidence.
- Search physically navigates to the answer.
- Agent activity and tool capabilities can also be represented in the same graph.

## Technical implementation proof

The submission must point judges to:

- public commit history during the submission period;
- runnable offline application;
- source ingestion and graph generation;
- neural and tree layouts;
- ExiaL pulse parser and replay mode;
- FAPI capability visualization;
- safe Exil intent preview;
- project export/import;
- validation scripts and GitHub workflow;
- clear separation between current implementation and original historical components.

## UX scoring plan

The demo must avoid spending time on setup.

Recommended order:

1. Open directly on a polished demo map.
2. Pan and zoom for five seconds.
3. Search for a concept and activate Zoom to Answer.
4. Open the exact evidence node.
5. Switch to Pulse Mode and replay ExiaL activity.
6. Switch to FAPI mode and show capability nodes.
7. Preview a safe Exil command.
8. End on the full map and one-sentence vision.

## Impact case

Primary users:

- developers operating agent systems;
- teams with large documentation and conversation histories;
- researchers and educators;
- small organizations that need understandable AI workflows;
- consultants managing knowledge across multiple client projects.

Concrete benefit:

- less time searching through flat histories;
- stronger evidence traceability;
- visible agent handoffs;
- safer preview of intended operations;
- one interface for memory, communication and capability inspection.

## Novelty case

The main differentiator is not any single graph algorithm.

The novelty is the combination of:

- spatial semantic memory;
- exact source evidence;
- visible machine-to-machine pulses;
- canonical event replay;
- declarative intent previews;
- tool capability and budget representation;
- offline operation without mandatory API spending.
