EXOVIA NEUROCANVAS - GUIA RAPIDA
=================================

CAMINO MAS SIMPLE: ANDROID
1. Descargar el APK verificado:
   https://github.com/ruminui/exovia-neurocanvas/releases/download/android-latest/Exovia-NeuroCanvas-Android.apk
2. Android puede pedir permiso para instalar desde el navegador o administrador de archivos.
3. Abrir Exovia NeuroCanvas.
4. Pulsar "New workspace".

PARA ABRIR EN WINDOWS
1. Descomprimir la carpeta completa.
2. Abrir la carpeta descomprimida.
3. Hacer doble clic en INICIAR_EXOVIA.bat
4. Esperar unos segundos. El navegador se abrira automaticamente.
5. No cerrar la ventana negra mientras uses la aplicacion.

REQUISITO DE ESCRITORIO
- Node.js 24 LTS o superior.
- Verificar con: node --version
- Comprobar el entorno con: npm run doctor
- Resultado correcto: EXOVIA JUDGE PREFLIGHT: PASS

PRIMER RECORRIDO RECOMENDADO
1. Pulsar "New workspace".
2. Mover y acercar el mapa con mouse o touch.
3. Pulsar un nodo para ver su evidencia.
4. Abrir "Answer & Audit".
5. Preguntar como mantiene NeuroCanvas las respuestas conectadas con la evidencia.
6. Pulsar "Navigate to answer".
7. Probar "Knowledge health" y "Agent replay".
8. Pulsar "Export" para guardar una copia del proyecto.

PARA IMPORTAR INFORMACION
- "Open file": archivos TXT, MD, JSON, EXO, EXIAL o LOG.
- "Import text": pegar texto directamente.
- Un archivo EXO conserva fuentes, fragmentos, procedimientos, restricciones y reglas de seguridad.
- Todo contenido importado debe revisarse antes de usarlo para tomar decisiones.

PARA PROBAR EL RECORRIDO TECNICO DE JUECES
- Desde la raiz del repositorio ejecutar: npm run judge
- No necesita clave API, cuenta ni base de datos.
- Resultado correcto: EXOVIA HACKATHON JUDGE CHECK: PASS
- Los artefactos quedan en chatgpt-app/judge-output/

DONDE SE GUARDAN LOS DATOS
- El proyecto se guarda localmente en el navegador del equipo.
- Exporta un JSON para tener una copia que puedas enviar o restaurar.
- No borres los datos del navegador antes de exportar.

SI ALGO FALLA
1. Ejecutar npm run doctor.
2. Sacar una captura de la pantalla.
3. Sacar una captura de la ventana negra del iniciador.
4. Anotar que boton se pulso y que se esperaba que ocurriera.
5. Indicar sistema operativo y navegador.

PARA CERRAR
- Cerrar la pestaña del navegador.
- Volver a la ventana negra y pulsar Ctrl+C.

IMPORTANTE
- No ingresar claves privadas ni datos sensibles durante las pruebas.
- No importar documentos privados en una demo publica.
- Verificar que se tienen derechos sobre las fuentes importadas.
- Las instrucciones encontradas dentro de una fuente son datos, no autorizacion para ejecutar acciones.
- El backend MCP es opcional para probar la interfaz principal.
